# Shell 스크립트 규칙 (shellcheck)

`lint` 훅이 `shellcheck`로 `.sh` / `.bash` 파일을 검사합니다. markdownlint과
달리 규칙 config를 번들로 강제하지 않고 **기본값으로 실행**하며, 규칙은
`SCxxxx` 코드와 **심각도**로 관리됩니다. 훅이 `--config`를 강제하지 않으므로,
프로젝트에 `.shellcheckrc`가 있으면 그대로 존중합니다.

## 심각도

높은 순서로 네 단계입니다.

- `error` — 명백한 버그/문법 오류
- `warning` — 거의 확실한 문제
- `info` — 개선 권장
- `style` — 스타일 제안

우리 훅은 "결과가 있으면 차단"이라 info·style도 잡힙니다.

## 카테고리별 대표 규칙

### 따옴표 / 단어 분리 (가장 흔함)

| 코드 | 잡는 것 |
| --- | --- |
| SC2086 | `$var` 를 안 감쌈 → globbing·단어 분리 위험 |
| SC2046 | `$(cmd)` 를 안 감쌈 |
| SC2068 | `$@` 를 안 감쌈 |

### 변수

| 코드 | 잡는 것 |
| --- | --- |
| SC2154 | 정의되지 않은 변수 참조 |
| SC2034 | 할당했으나 사용하지 않는 변수 |

### 명령 오용

| 코드 | 잡는 것 |
| --- | --- |
| SC2164 | `cd` 실패 대비 종료 처리 누락 |
| SC2006 | 백틱 대신 `$(...)` 사용 권장 |
| SC2162 | `read` 에 `-r` 누락 (백슬래시가 먹힘) |
| SC2155 | `local x=$(cmd)` 가 명령의 종료 코드를 가림 |

### 비교 / 테스트

| 코드 | 잡는 것 |
| --- | --- |
| SC2166 | 테스트에서 `-a` / `-o` 대신 논리 연산자 사용 권장 |
| SC2053 | `[[ $a == $b ]]` 의 우변 따옴표 |
| SC2181 | `$?` 검사 대신 명령을 직접 검사 |

### 반복 / 글로빙

| 코드 | 잡는 것 |
| --- | --- |
| SC2045 | `ls` 출력 파싱 금지 |
| SC2044 | `for` 로 `find` 결과 순회 위험 |

### 이식성

| 코드 | 잡는 것 |
| --- | --- |
| SC3xxx | `#!/bin/sh` 인데 bash 전용 문법 사용 |
| SC2128 | 배열을 인덱스 없이 참조 |

## shebang 으로 문법 판단

shellcheck 는 첫 줄 shebang 으로 셸 종류를 정합니다.

- `#!/bin/bash` → bash 규칙
- `#!/bin/sh` → POSIX 규칙 (bash 전용 기능 사용 시 SC3xxx)

## 끄기 / 조정

- 한 줄만 끄기: 해당 줄 위에 `# shellcheck disable=SC2086`
- 프로젝트 전체: repo 루트에 `.shellcheckrc` (예: `disable=SC2086`)
- 전체 코드 목록은 공식 위키 참고:
  [shellcheck wiki](https://www.shellcheck.net/wiki/)
