# keyword-docs-new-docs

`context` 모듈의 keyword-docs 계열에 넣을 **문서를 템플릿으로 생성하는 절차**.
문서 파일 생성과 인덱스 등록을 한 번에 끝낸다 — 등록 단계는
[keyword-docs-add-index](../keyword-docs-add-index/SKILL.md) 절차를 그대로
따른다. 파일 저장과 인덱스 수정은 반드시 **사용자 승인 후에만** 한다.

인자:

- `[카테고리]` — `templates/` 아래 파일명과 1:1 (`msg-format`, `db-schema`,
  `domain`). 없으면 목록을 보여주고 물어본다.
- `[이름]` — 문서 대상 (커맨드명 / 테이블명 / 용어). 없으면 물어본다.

## 1. 템플릿 읽기

이 스킬 디렉토리의 `templates/<카테고리>.md`를 Read로 읽는다. 템플릿은 3층
구조다:

- **메타 주석** (`<!-- ... -->`) — 저장 경로 관례, 필수/선택 슬롯, 키워드
  기본값, precision 기본값. 스킬만 읽고 **생성된 문서에서는 제거**한다.
- **골격** — `제목 → 한 줄 요약 → 핵심 표 → 주의 → --- → 상세` 순서 고정.
  주입은 앞 `maxCharsEach`(기본 1200자)만 들어가므로 핵심이 `---` 위에 오게
  이미 설계되어 있다 — **구조를 재배치하지 않는다**.
- **슬롯** (`{{...}}`) — 값 이름이 아니라 **채우기 지시문**이다. "없으면 삭제"
  표시가 있는 줄은 해당 없을 때 통째로 지운다.

## 2. 슬롯 채우기 — 소스 선택

AskUserQuestion으로 소스를 정한다:

- **코드에서 추출** (기본 제안) — 대상 이름으로 코드를 검색해(메시지 정의,
  DDL/엔티티, 도메인 로직) 필수 슬롯을 채운다. 근거로 삼은 파일을 같이
  보여준다. 코드에 없는 슬롯만 질문한다.
- **직접 입력** — 필수 슬롯만 하나씩 질문한다. 선택 슬롯은 "생략 가능"을
  명시하고 물어본다.

## 3. 초안 승인 → 저장

1. 메타 주석을 제거하고 슬롯을 모두 채운 **초안 전문**을 보여주고 승인받는다.
2. 저장 경로는 메타의 관례를 따른다 (예: `.claude/docs/msg/cmd-start-lot.md`).
   생성 문서는 주입 전용 산출물이므로 인덱스·설정과 함께 `.claude/` 아래에
   모은다 (add-index가 기존 문서를 제자리에 두고 등록하는 것과 대비).
   디렉토리가 없으면 만들고, **같은 경로에 파일이 이미 있으면 덮어쓰지 않고**
   사용자에게 확인한다.
3. markdownlint를 통과해야 한다 (lint 훅이 검사한다) — 표 파이프 양옆 공백,
   리스트 앞뒤 빈 줄.

## 4. 인덱스 등록 — add-index 절차 재사용

저장이 끝나면 keyword-docs-add-index의 §2~§7을 그대로 수행한다. 차이점만:

- 카테고리(인스턴스)는 이미 정해져 있으므로 §2는 확인만 한다 — 해당 인스턴스가
  `.claude/context.json`에 꺼져 있으면 알리고 켤지 물어본다.
- 키워드 제안(§3)은 템플릿 메타의 **키워드 기본값**에서 출발한다 (예:
  msg-format이면 커맨드명 소문자). 충돌 검사·precision·승인 후 append는 동일.

## 5. 일괄 모드 (선택)

"커맨드 전부 문서화" 같은 요청이면: 대상 목록을 먼저 확정받고, 초안 생성은
**haiku급 subagent를 병렬**로 돌린다 (기계적 추출 작업 — 토큰 절약). 생성된
초안들을 한 번에 보여주고 일괄 승인 후 저장·등록한다. 인덱스 append는 충돌
검사를 위해 **직렬로** 한다.

## 새 카테고리 추가

`templates/<이름>.md` 하나 추가하면 끝 — 카테고리 인자가 템플릿 파일명과
1:1이므로 스킬 본문은 수정할 필요 없다. 단 대응하는 프로바이더 인스턴스가
없으면 `core/context/lib/providers/`에 `makeKeywordDocsProvider` 인스턴스와
레지스트리 한 줄을 같이 추가한다.

## 한계

- 슬롯 내용의 정확성은 소스(코드/사용자 입력)에 달려 있다 — 이 스킬은 구조와
  절차만 보장한다. 코드 추출 초안은 반드시 사용자 검토를 거친다.
- 카테고리 추정을 하지 않는다 — add-index와 달리 카테고리를 인자로 받는 것이
  의도된 설계다 (템플릿이 카테고리를 결정하므로).
