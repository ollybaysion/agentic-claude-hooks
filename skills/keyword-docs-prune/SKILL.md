# keyword-docs-prune

`context` 모듈의 keyword-docs가 쌓은 주입 stats(오탐 프루닝 layer 1)를 분석해
**어떤 키워드를 지우거나 좁혀야 하는지** 제안하고, 승인받아 적용하는 절차.
기계는 집계·판정·제안까지만 — **삭제 확정은 항상 사람이 한다.** 자동 삭제 금지.

인자 `[days]`: 분석 기간(기본 30). 예: `/keyword-docs-prune 14`.

## 1. stats 파일 찾기

- 위치: `$CLAUDE_CONTEXT_STATS_DIR` (미설정 시 `~/.claude/context-stats/`) 아래
  `<sha256(cwd) 앞 16자>.jsonl`. 해시는 다음으로 계산한다:

  ```bash
  node -e 'console.log(require("crypto").createHash("sha256").update(process.cwd()).digest("hex").slice(0,16))'
  ```

- 파일이 없거나 비어 있으면: keyword-docs가 이 프로젝트에서 켜져 있는지
  (`.claude/context.json`), 주입이 실제로 있었는지 확인하라고 안내하고 **종료**.
- 한 줄 형식: `{ "ts": ms, "session": "...", "keywords": ["발화한 것들"], "path": "문서", "mode": "full"|"link" }`
  (`mode: "link"`는 한 줄 포인터만 주입된 것 — 오탐이어도 비용 ~20토큰이라 처방
  우선순위가 낮다. 절감 계산 시 full ~400–600토큰과 구분할 것.)

## 2. 키워드 단위 집계

기간(`days`) 안의 라인만 대상으로, **키워드별로**: 발화 횟수 · 대상 문서 ·
고유 세션 수 · 최초/최근 발화 시각. (한 라인의 `keywords` 배열 각 원소가 1발화.)
jq나 node 한 줄로 집계하되, 결과는 표로 정리한다.

## 3. 후속 참조 판정 (소급 layer 2 — 가능한 범위에서)

발화 상위 키워드에 대해, stats의 `session`으로 트랜스크립트를 조인한다:

- 트랜스크립트: `~/.claude/projects/<sanitized-cwd>/<session>.jsonl`
  (sanitized-cwd = cwd의 `/`를 `-`로). 보존기간(기본 30일)이 지난 세션은 조인
  불가 — **그런 세션은 "판정 불가"로 표기하고 오탐으로 세지 않는다.**
- 주입 이후(ts 기준) 그 세션에서 다음이 있었는지 본다:
  - 그 문서 경로를 `Read` 했다 → **강한 유관**
  - 응답/도구 호출이 그 문서 경로나 고유 용어를 언급했다 → **유관**
  - 아무것도 없다 → 무참조 1회
- 트랜스크립트가 크면 해당 세션 파일에서 문서 경로 문자열을 `rg`로 찾는 것으로
  충분하다 (주입 라인 자체는 제외하고 셀 것 — 주입 텍스트에 경로가 포함된다).

## 4. 리포트 + 처방

키워드별 표: `키워드 | 문서 | 발화 | 세션 | 후속참조율 | 처방`. 처방 규칙:

| 패턴 | 처방 |
| --- | --- |
| 발화 잦음 + 후속 참조 거의 없음 | **제거** 또는 더 좁은 phrase로 교체 (예: `mcp` → `mcp server`) |
| 발화 잦음 + 참조가 가끔 있음 (지우긴 아까움) | **`precision: 0.5` 강등** — 전문 대신 한 줄 포인터(~20토큰)만 주입, 관련 있으면 모델이 직접 Read |
| 발화 잦음 + 참조도 잦음 | 유지 (정상 동작). `mode: "link"`인데 참조가 잦다면 역으로 `precision: 1` 승격 고려 |
| 발화 0회 | 무해(비용 0) — 인덱스 정리 차원에서만 제거 고려 |
| 판정 불가(트랜스크립트 만료)가 대부분 | 결론 유보 — 기간을 줄여 재분석 제안 |

오탐 1회의 비용은 대략 그 문서 슬라이스 토큰(~400–600)이다. 처방에 예상 절감량을
같이 적으면 판단이 쉬워진다.

## 5. 승인 후 적용

1. 처방을 보여주고 **어느 항목을 적용할지 사용자에게 묻는다** (AskUserQuestion 또는
   명시적 확인). 승인 없는 수정 금지.
2. 승인된 항목만 `<project>/.claude/context-docs.json`에서 키워드 삭제/교체
   (엔트리의 마지막 키워드를 지우면 엔트리 자체를 제거할지도 함께 확인).
3. 훅이 매 턴 인덱스를 다시 읽으므로 **저장 즉시 적용** — 리로드 불필요.
4. 적용 내역을 한 줄씩 요약해준다 (무엇을 왜 지웠는지).

## 한계 (분석 시 정직하게 표기)

- 후속 참조는 프록시다 — "참조 없음 ≠ 무조건 오탐"(읽지 않고도 유용했을 수 있음).
  그래서 처방은 누적 경향으로만 내고, 확정은 사람이 한다.
- dedup으로 억제된 매치는 기록에 없다(토큰을 안 썼으므로 오탐 후보가 아님).
- stats는 주입 사실만 안다 — 주입 **품질**은 트랜스크립트 조인으로만 근사된다.
