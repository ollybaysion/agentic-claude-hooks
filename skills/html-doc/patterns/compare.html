목차

    비교/결정 패턴 견본

    html-doc 스킬 · patterns/compare.html — 이 파일이 복사 원본이다 (비교/결정 패턴 CSS:START…END + 데모 마크업)

    1. 언제 쓰나

    설계 문서의 대안 검토 섹션 — 옵션 2~4개를 나란히 놓고 채택/기각/보류를 판정하고, 결정과 근거를 날짜와 함께 기록할 때. 기준이 여러 축이면 옵션 카드 아래에 기준 표(.table-wrap)를 병기한다.

    블록 4종: compare(옵션 카드 그리드) · verdict(판정 배지 v-ok/v-bad/v-hold) · pl(+/− 장단점 목록) · decision(결정 콜아웃 — 날짜·근거 필수).

    2. 데모 — 에이전트 조회 채널 결정

    기각

    MCP 서버로 노출

    • CC와 표준 결합 — 도구 스키마 자동
    • 소비자가 CC로 한정 — 타 에이전트·배치가 못 씀
    • 사내 방화벽 구성에서 프로세스 수명 관리 복잡
    채택

    HTTP + JSON (POST /agent/query)

    • 모든 소비 채널 대응 — CC·타 에이전트·curl
    • 코어 runAgent 재사용 — 신규 코드 최소
    • 도구 스키마를 계약 문서로 별도 유지해야
    보류

    gRPC 스트리밍

    • 토큰 스트리밍에 유리
    • 사내 소비자 전부 HTTP 우선 — 수요 미확인
    결정 · 2026-07-17
    HTTP+JSON 채택. 근거: 에이전트 조회 채널의 소비자가 CC 하나가 아니라는 것이 확정적 — MCP는 그 하나에만 최적화된 결합이다. 로컬 fixture로 증명 후 진행.

    3. 사용법 (모델용 절차)

    1. CSS 블록을 문서 <style> 끝에 복사(판정 토큰 포함 — 다른 패턴과 중복돼도 무해).
    2. 마크업 블록을 복사해 교체. 채택안에만 chosen 클래스, 배지는 카드당 하나.
    3. decision 콜아웃엔 날짜와 근거를 반드시 — "무엇을" 만이 아니라 "왜"가 기록의 가치다. 결정이 아직 없으면 콜아웃을 빼고 전부 v-hold로.
    4. 렌더 후 check.mjs는 평소대로.
