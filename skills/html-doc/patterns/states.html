목차

    상태 전이 패턴 견본

    html-doc 스킬 · patterns/states.html — 이 파일이 복사 원본이다 (상태 전이 패턴 CSS:START…END + 데모 마크업)

    1. 언제 쓰나

    대상이 상태 몇 개와 전이 규칙으로 정의될 때 — 신뢰 티어, 워크플로우 단계, 요청 라이프사이클. 스트립은 주 경로 요약이고 전이 표가 진실원이다: 스트립에는 대표 경로 하나만 가로로 놓고, 분기·역행·예외 전이는 표에만 적는다. 전이가 서로 얽히는 복잡 상태기계는 이 패턴 밖 — 표만 쓰거나 pre.diagram으로.

    블록 3종: states(상태 필 스트립, initial=점선 · 성격색 s-ok/s-hold/s-bad) · tr(전이 라벨) + 전이 표(.table-wrap 재사용 — from/to/트리거/가드).

    2. 데모 — 티어드 값 신뢰 전이

    scaffold ─ apply → inferred ─ promote(사람 전용) → confirmed ─ 고아화 → deprecated
    fromto트리거가드
    scaffoldinferredenrich 제안 applyenrich-cli만 쓰기 경로
    inferredconfirmedpromote사람 전용 · If-Match rev
    confirmedinferred행 수정수정 시 자동 강등
    (모든 상태)deprecated슬롯 고아화자동 — 주입 제외
    deprecatedinferred복원confirmed 직행 금지

    3. 사용법 (모델용 절차)

    1. CSS 블록을 문서 <style> 끝에 복사(판정 토큰 포함).
    2. 스트립에는 대표 경로 하나만. 초기 상태는 initial (점선), 성격색은 안정 s-ok/과도 s-hold/이탈 s-bad — 색이 없는 상태(기본 회색)도 허용.
    3. 전이 표가 사양이다: 분기·역행 포함 모든 전이를 from/to/트리거/ 가드로 적는다. 스트립과 표가 어긋나면 표가 맞다.
    4. 렌더 후 check.mjs는 평소대로.
