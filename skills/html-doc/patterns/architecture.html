    1. 언제 쓰나

    계층·경계·파이프라인으로 표현되는 시스템 구조 — 네트워크 경계 안의 컴포넌트, 층 사이의 데이터 흐름, 단계 처리 라인. 위→아래(존 사이)와 왼→오른쪽(파이프)만으로 그린다 — 좌표 계산이 없다.

    안 되는 것: 자유 배치 박스 사이를 대각선·우회 엣지가 잇는 임의 그래프. 그건 pre.diagram(ASCII)으로 후퇴하거나, 구조를 계층형으로 다시 정리한다 — 엣지가 얽히는 그림은 대개 설계가 덜 정리됐다는 신호다.

    블록 4종: zone(경계, 중첩 가능, tone-a/b 소속색) · node(컴포넌트, hi 강조) · pipe(존 안 가로 단계) · flow(존 사이 수직 화살표+라벨).

    3. 사용법 (모델용 절차)

    1. CSS 블록을 문서 <style> 끝에 복사. 존 색은 주제에 맞게 바꿔도 되나 라이트/다크 쌍 유지.
    2. 마크업 블록을 복사해 교체. 존 = 경계 하나, 그 안에 node-row(나열) 또는 pipe(순서 있는 단계). 존 사이 데이터 이동은 flow로.
    3. 다이어그램 내부에 h2/h3 금지(.name·.zone-label 사용). 색이 소속을 뜻하면 본문에 한 줄 범례를 쓴다.
    4. 렌더 후 check.mjs는 평소대로.
